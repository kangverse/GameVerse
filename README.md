# GameVerse

**GameVerse** is a large-scale, long-horizon gameplay video dataset designed for world modeling, controllable video generation, and camera-aware video understanding. It contains **188K high-quality clips** (≈ **3,115 hours**), aligned with **validated camera trajectories** and **multi-granular structured captions**, sourced from over **110 AAA and indie game titles** across both first- and third-person perspectives.

This repository hosts the released metadata and per-clip annotations of the dataset:

- Raw **YouTube URLs** of the source videos.
- Per-clip **fine-grained structured captions**.
- Per-clip **camera pose** sequences estimated by ViPE.

The corresponding video frames are not redistributed here; they can be reproduced from the released URLs via the construction pipeline described below.

## Repository Structure

| Path                  | Description                                                                                                                                                         |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `gameverse_url.txt`   | Raw YouTube URLs (one per line) of the **11,002 source long-form gameplay videos** scraped from YouTube. These are the seeds from which all clips are derived.       |
| `caption/`            | Per-clip **multi-granular structured captions** in JSON. Each file is named `<youtube_id>_<start_us>_<end_us>.json` and corresponds to one segmented clip.           |
| `pose/`               | Per-clip **camera pose** sequences (intrinsics + per-frame extrinsics) generated by [ViPE](https://github.com/nv-tlabs/ViPE) and validated for stability and drift. |
| `README.md`           | This file.                                                                                                                                                          |

### Clip naming convention

Each clip ID has the form `<youtube_id>_<start_microseconds>_<end_microseconds>`, e.g. `0OYGmgrn-PQ_0010950_0014557`, which uniquely identifies a temporal segment of a source YouTube video. The same ID is shared across the `caption/` and `pose/` folders so annotations can be joined directly by filename.

### Caption schema

Each caption JSON contains four complementary semantic layers, plus segment-level annotations:

- **Global static** (`overall`): scene-level context — terrain, weather, time of day, lighting, viewpoint, camera motion, architecture style, etc.
- **Local static** (`segments[*].scene_state` / `elements`): functional zones, landmarks, and visible objects within each temporal segment.
- **Dynamic events** (`segments[*].events`): agent-independent world-state changes (scene transitions, motion changes, environmental events).
- **Agent actions** (`segments[*].agents.ego_action`, `notable_agents`): the main subject's behavior under a unified action vocabulary, plus pedestrian and vehicle agents.

Captions also include `merged_segments` for hierarchical conditioning and an `annotation_meta` block recording the VLM model and processing details.

## Dataset Construction

GameVerse is built by a unified pipeline that turns a curated pool of long-form gameplay videos into structured, minute-scale data units. The pipeline addresses three limitations common to prior gameplay datasets: fragmented short clips, unreliable or missing camera poses, and flat semantic annotations that conflate static context with dynamic events.

### 1. Data Collection

Source videos are collected from **YouTube**, the only open source that simultaneously offers long-form gameplay across many titles, both FPV and TPV camera schemes, and a wide range of player behaviors. Videos are retained at the highest available quality (up to **1080p / 30–60 FPS**). The raw pool contains **11,002 videos**, totaling **4,692 hours (≈195.5 days)** and approximately **986.6M frames**, drawn from open-world AAA and indie titles chosen to span both perspectives and the eight target behavior categories: *exploring*, *combat*, *walking*, *running*, *driving*, *interacting*, *climbing*, *idle*.

### 2. Scene-consistent Segmentation

Long-form videos are split into minute-level **scene-consistent** segments (up to **120 seconds**) so that long-horizon temporal continuity is preserved:

- **Hard cuts** are detected with [TransNetV2](https://github.com/soCzech/TransNetV2) using frame-level transition probabilities.
- **Soft transitions** (panning, UI state changes) are detected via CLIP-based cosine semantic distance between sampled frames; large jumps mark smooth-transition boundaries.

After segmentation we obtain **272,128 clips** (**4,661 hours**), of which the 120-second clips alone contribute about **2,773 hours**.

### 3. Multi-stage Video Quality Evaluation

Raw gameplay clips can still contain subtitles, watermarks, HUD overlays, compression artifacts, near-duplicates, and unstable temporal dynamics. A multi-stage filter is applied:

1. **Basic filtering** — drop low-resolution or corrupted videos.
2. **Content/quality assessment** — OCR-based detection of subtitles, HUDs, and overlays.
3. **Stability and UI detection** — flag unstable motion patterns and temporal artifacts.
4. **Deduplication and diversity selection** — remove near-duplicates while preserving scenario diversity.
5. **Temporal refinement** — score temporal consistency with [COVER](https://github.com/vqassessment/COVER) and [VBench](https://github.com/Vchitect/VBench).

The final clip score is a weighted sum across stages; only clips passing every constraint are retained.

### 4. Camera Pose Estimation and Validation

Camera intrinsics and per-frame extrinsics are estimated with **[ViPE](https://github.com/nv-tlabs/ViPE)**. Because pose estimation in dynamic gameplay is inherently noisy, every trajectory is validated against three error modes:

- **Sudden jumps** — large non-physical pose changes between consecutive frames (tracking failures, relocalisation errors).
- **High-frequency jitter** — local instability producing unnatural oscillations.
- **Long-term drift** — error that accumulates over time.

A unified trajectory-quality score combines local relative-pose increments (jumps, jitter) with a global smoothness term (drift), and threshold-based filtering removes unreliable samples. The resulting per-clip poses are released under `pose/`.

### 5. Multi-granular Structured Captioning

Rather than producing a single flat caption per clip, GameVerse decomposes each clip's semantics into **four complementary layers** with a vision-language model (Kimi-K2.5):

- **Global static** — overall environment and scene context (terrain, weather, lighting, viewpoint).
- **Local static** — local scene configurations within a temporal segment (functional areas, landmarks, visible objects).
- **Dynamic events** — agent-independent world-state changes (crowd movement, traffic, environmental transitions).
- **Agent actions** — the main subject's behavior under a unified action vocabulary that disentangles action from viewpoint and supports controllable generation.

This structured form is much richer than single-line captions and can be consumed directly as a hierarchical conditioning signal.

## Dataset Statistics

After the pipeline above, GameVerse contains **188K high-quality gameplay clips**, totaling **3,115 hours**, of which **1,747 hours** are continuous two-minute clips.

### Scale and long-horizon distribution

Unlike prior gameplay datasets dominated by clips shorter than ten seconds, GameVerse preserves substantial temporal continuity. Over **52K** clips reach the **120-second** upper bound and together contribute **≈1,747 hours** of continuous gameplay video — to our knowledge, an order of magnitude more continuous footage than any previously released gameplay corpus. Clips of 60 s or longer account for a further large portion of the dataset and provide the minute-scale supervision required to push interactive world-model rollouts beyond the typical one-minute coherence ceiling.

### Interaction and environment diversity

- **110+ distinct game titles**, with a long-tail genre composition. Top titles (e.g. *God of War*, *Red Dead Redemption 2*, *Ghost of Tsushima*, *Elden Ring*, *The Witcher 3*) each contribute **1,800+ clips**; the tail extends across dozens of additional AAA and indie titles.
- **Camera perspective**: ≈ **24%** first-person, **73%** third-person follow camera, **3%** specialised (drone, vehicle-mounted, gimbal-follow, stationary). The 1:3 FPV-to-TPV ratio supplies substantial supervision for both settings.
- **Behavior categories**: *exploring*, *combat*, *walking*, *driving*, *interacting*, *climbing*, *running*, *idle*, with exploring, combat, walking, and driving contributing the largest share of total duration.
- **Scene attributes**: urban, rural, forest, highway, temple, and industrial environments under varied architectural styles, weather, time of day, season, and crowd-density levels.

### Camera trajectory diversity

Camera motion is grouped into *static*, *rotational*, *translational*, and *complex* patterns:

- **Forward motion**: **34.7%** — the largest share, reflecting continuous navigation and exploration.
- **Complex motions** (translation coupled with rotation): **22.0%** — particularly challenging for long-horizon modeling because they demand simultaneous geometric and identity consistency under non-trivial viewpoint change.
- Lateral movement, orbit, and mixed rotational patterns are also well represented.

This is a markedly richer trajectory distribution than walking-video corpora dominated by forward translation, and is not restricted to a single title's controller scheme.

### Caption annotation statistics

Across the four semantic layers, caption length differs by design:

- **Local static** — median length **≈ 895 characters**.
- **Dynamic events** — median length **≈ 775 characters**.
- **Global static** — concise high-level scene context.
- **Agent actions** — drawn from a constrained vocabulary so the controllable channel can be modulated cleanly during conditional generation.

Vocabulary analysis shows the four layers concentrate on largely disjoint sets of words — scene attributes, spatial layouts, environment transitions, and explicit motion — i.e. the schema produces semantically separated annotations rather than overlapping rephrasings of the same clip.

## Reproducing video frames from URLs

This repository releases URLs and annotations only. To reconstruct a clip:

1. Download the source video from the URL in `gameverse_url.txt` at its highest available quality.
2. Re-run the segmentation step (TransNetV2 + CLIP-based smooth-transition detection) **or** use the start/end microsecond fields encoded in the clip filename to slice the source video directly.
3. Pair each resulting clip with its `caption/<clip_id>.json` and `pose/<clip_id>.*` annotation file.
